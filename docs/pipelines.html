250/384(65%) — 신뢰 가능 11개, 4축(안정성/정밀도/재현성/속도) 벤치마크 재현성 100회 반복까지 완료. 1위 step-3.5-flash(0.866), 하위 3개(deepseek-v4-pro·glm-5.2·minimax-m3)는 재현성이 아니라 대량 단일모델 발사로 인한 인프라 붕괴(모델별 총량 쿼타 버킷 소진 — mistral-large-3 외 추가 확인)로 최종 마감.

D48(전신), D56~D116

왜 16개 후보 모델 중 일부만 동작했나 — 감쇠 분석 (16 → 신뢰 가능 5)

파이프라인 03의 벤치마크는 16개 모델 전부를 동일한 하니스 · 동일한 7키 풀 · 동일한 중앙 설정(timeout 600s, max_tokens 2048)으로 돌렸다. 그런데도 성공률이 0%~95.8%로 갈린 이유는 우리 코드가 아니라, 모델별 규격(tool-calling) 준수도와 NVIDIA Build 무료 티어 서빙 인프라의 차이였다 — 전 건 원인 진단 완료(D100~D104).

  • 신뢰 가능 · 6개 qwen3-next-80b · step-3.5-flash · mistral-medium-3.5 · nemotron-super-49b-v1.5 · llama-4-maverick · mistral-large-3 (87.5~100%, 질문생성 역할 기준) — nemotron과 mistral-large-3는 원래 0%대였다가 우리 쪽 설정 원인(max_tokens 기아)을 고쳐 각각 95.8%·100%로 회복한 사례. 단 mistral-large-3는 채점기 역할에선 퇴행 생성 루프(캡 2048/4096/8192 전부 예산 전액 소진, 가시 출력 ~900자)로 부적합 판정 — 4축 벤치마크의 안정성 축(질문생성×채점 end-to-end)에 0점으로 반영(D108).
  • 모델 자체 결함 · 2개 gpt-oss-120b(0%) · gpt-oss-20b(4%) — tool_choice 강제를 무시하고 자유서술로 응답. 스키마 강제 tool-calling 기반인 이 파이프라인에선 구조적으로 사용 불가 (D66/D80/D89부터 일관 재현).
  • NVIDIA 서버측 · 2개 llama-3.3-70b(0%) — HTTP 503 워커풀 고갈, NVIDIA 포럼 교차검증, 최종 제외. mistral-nemotron(4%) — 클라이언트 타임아웃을 450초로 줘도 ~302초에 NVIDIA 게이트웨이가 먼저 끊음(504), 최종 제외.
  • 계정 접근권한 · 1개 kimi-k2.6(0%) — 카탈로그(/v1/models)엔 있지만 채팅 엔드포인트가 HTTP 404. preview 권한 문제로 추정.
  • D110 전원 회복 · 5개 deepseek-v4-pro 42%→100% · minimax-m3 71%→96% · nemotron-3-super-120b 29%→96% · glm-5.2 13%→88% · qwen3.5-122b 58%→83% — 개별 진단(사용자 지시) 결과 실패 69건이 3클래스(429=우리 버스트 잔재 / content=None=512 기아 / content에 JSON=기아·미준수 혼합)로 갈렸고, 현재 설정(4096 캡·12rpm)으로 재실행하자 60/69 회복. "진짜 천장 미상"이던 티어가 통째로 신뢰 티어에 합류(신뢰 11개, 전체 250/384=65%). 잔여 9건은 인프라 노이즈 2 + qwen3.5 고유 한계 3 + glm 간헐 미준수 4.

핵심 교훈: "낮은 성공률 = 나쁜 모델"이 아니다. 16개 중 우리 설정이 원인이었던 게 2개(둘 다 근본수정 완료), 무료 티어 인프라가 원인인 게 4개, 모델 결함이 2개 — 라이브 벤치마크에서 원인 분리 없이 순위만 읽으면 nemotron(0%→95.8%)·mistral-large-3(4%→100%) 같은 모델을 억울하게 탈락시키게 된다. 최종 순위는 4축 벤치마크(안정성=질문생성×채점 end-to-end · 정밀도(자기채점) · 재현성 100회 반복 · 속도, D116 마감): step-3.5-flash 0.866 > mistral-medium-3.5 0.749 > qwen3-next-80b 0.742 > nemotron-3-super 0.663(범위 밖 점수 출력 — 100회가 3회보다 이 결함을 더 정확히 잡음) > qwen3.5-122b 0.589 > nemotron-49b 0.531 > deepseek-v4-pro 0.487(총량 쿼타 버킷 소진 — 몇 시간 전엔 100%) > maverick 0.248(서빙 장애) > mistral-large-3 0.229(채점기 퇴행 루프) > glm-5.2 0.104(총량 쿼타 버킷 소진) > minimax-m3 0.1(DEGRADED 재발). 하위 3개(deepseek·glm·minimax)는 100회 재현성 측정 중 한 모델에 30~40분간 2000+콜을 몰아치자 무너진 것 — 재현성이 아니라 인프라 붕괴이고, mistral-large-3에서만 봤던 "모델별 총량 쿼타 버킷"이 더 넓게 존재한다는 새 발견이다. 정밀도는 채점 성공 모델 전원 1.00 — 승부처는 재현성(temp=0인데도 0~0.615)과 채점기 역할 안정성이다. 상세 근거는 README D94~D116 참고.
