# Depth Ladder 7단계 강제 템플릿

# D1: 7단계 전부를 필수 필드로 고정
#   WHY: 이전 시뮬레이션(Study-Match-)에서 항목①(Competitions.tsx)만 우연히 7단계를 다 채웠고
#        ②③④는 즉흥적으로 1~2단계만 생성되어 피드백 품질이 '중'에 그침 — 즉흥 생성은 편차가 재현 불가능한 수준으로 큼
#   COST: 매 finding마다 7개 질문을 다 만들어야 해서 피드백 블록의 생성 비용이 늘어남 (판단 블록에서 우선순위 상위로 걸러진 항목에만 적용해 상쇄)
#   EXIT: 7단계가 과하다고 판단되면 What/Why/Reflection 3단계 축소판으로 다운그레이드, 필드명은 그대로 유지해 하위호환

각 finding은 판단 블록을 통과하면 아래 7개 필드를 전부 채워야 "피드백 완료"로 간주한다. 하나라도 비면 미완료로 표시.

| 단계 | 질문 목적 | 채우지 않으면 생기는 문제 |
|---|---|---|
| 1. What | 무엇을 했는지 사실 확인 | 이후 질문이 헛다리 짚음 |
| 2. How | 구체적 구현 메커니즘 확인 | "말로만 아는 척" 통과 가능 |
| 3. Why | 선택 이유 | 판별력의 핵심, 생략 금지 |
| 4. Alternative | 대안 인지 여부 | 대안을 모르면 "왜"에 답할 수 없다는 걸 드러냄 |
| 5. Trade-off | 선택지 간 비교 | 표면적 정답과 진짜 이해를 가르는 지점 |
| 6. Constraint | 제약조건(시간/기술/팀) 반영 여부 | 의도적 선택 vs 어쩔 수 없는 선택 구분 |
| 7. Reflection | 자기 수정 | Ownership Cycle의 핵심 차별점(기존 소크라테스식엔 없던 단계) |
